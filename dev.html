Mylar Developer Documentation

Mylar is in a pre-release stage, evolving rapidly, and provides no fixed APIs.  We expect to announce some stability in the APIs before the end of summer, and until then the policy is extend at your own risk. If you are interested in extending Mylar please email mylar-dev@eclipse.org.

mylar workspace

Mylar requires Eclipse SDK 3.1 and JDK 1.5.  To set up a workspace for mylar development:
- Add CVS location: dev.eclipse.org /home/technology
- Check out all of the projects under org.eclipse.mylar

bugzilla & tasks components

Decoupled from the rest of Mylar and can be built independently:
- Add CVS location as above
- Check out org.eclipse.mylar: bugzilla.core, bugzilla.ui, bugzilla.test, tasks, core

monitor component

Used for running the user study, the Mylar monitor records, obfuscates, and uploads usage statistics to a UBC server.  To assure the integrity of the study data the sources are on a non-public UBC server for the duration of the study. If you have a UBC account you can access:
- Add CVS location: cascade.cs.ubc.ca /isd/se2/project/mylar/cvs
- Check out all of the org.eclips.mylar.monitor projects

conventions

Bugzilla
- Priorities: P1: do this week, P2: should do this week, P3: considered for next week, P4: later, P5: requires usage input to be prioritized.
- Releases: for the duration of the study, a stable minor-minor version number rev happens each Friday, and minor rev on the first Friday of each month.
- To create a patch, right-click the corresponding project and select Team -> Create Patch.  Repeat for each project, naming the patch accordingly (e.g. patch-core.txt, patch-core.tests.txt). Please submit a unit test along with the patch.

Building and sources
- With the default Eclipse 3.1 compile options there should be no warnings after a build.
- For handling exceptions and logging use MylarPlugin.log(..) or MylarPlugin.fail(..)
- In CVS, files ending with .mmap are MindManager mind maps (there is a free viewer for Windows).
